# No Em Dashes Rule (optional style rule)

This repo ships with a strong preference against em dashes, inherited from the original author. Keep it,
change it, or delete this file. It is a style choice, not a requirement for the system to run.

## The rule

Avoid the em dash character in anything the system produces: research summaries, Notion pages, content
drafts, file names, and reports. Em dashes read to many people as generic AI output.

## What to use instead

- A period (start a new sentence). Usually the cleanest fix.
- A comma (for a pause or an aside).
- A colon (when introducing a list or an explanation).

For number ranges, write "1 to 5" or use a plain hyphen ("1-5"). For money, write "1,500 to 2,000".

## If you do not care about this

Delete this file. Nothing in the content-research command depends on it.
